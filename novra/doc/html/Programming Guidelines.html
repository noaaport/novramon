Novra Device Communications Library

Programming Guidelines

This section of the documentation provides a rough guideline on how to use the Novra Device Communications API.  The general steps to follow are described below.

  1. Use the function discoverDevices() to fill in a Novra_DeviceList structure listing Novra devices on the network.  This will turn up all Novra devices that are broadcasting status on the local subnet using one of the auto_detect_status_ports.  This step is optional, but if you don't do this, or if your device is not using one of the auto detect ports, then you will need to fill in aNovra_DeviceEntry yourself before trying to open a device handle.
  2. Open a handle to the device of your choice.  You can do this by passing one of the elements of the Novra_DeviceList to the function openDevice(), or by filling in your own Novra_DeviceEntry.  You need to use the second method if the device is not using one of the auto-detect status ports. Some devices support the openRemoteDevice() function, which allows communication with a device not on the host subnet.
  3. If your device cannot be opened using openDevice(), the function forceDevice() can be used to force it back into a state where you can communicate with it, but the current network settings on the device will be replaced with defaults. Some devices also support the function forceRemoteDevice() which can be a very dangerous function if misused.
  4. The type of device can be determined using the function getDeviceType().
  5. Some devices require a password in order to communicate with them. The function specifyPassword() is used to provide a password entry that the receiver will use to verify against its stored password in order to authorize command messages. This only needs to be done once ... after the device has been opened.
  6. Use the function configureNetwork() to change the network settings of the device.  The function GetAvailablePort() can be used to choose a destination status port from the list of auto-detect status ports that is not already in use by the host computer.
  7. Use the function configureRF() to tune the device.
  8. Use he function setPIDList() to select content.
  9. If your device is an S75CA or S75CA Pro , the function setProgramList() can be used to select the programs to be sent to the CAM. For these devices, there is also a function, setCAMWatchdogTimeout(), for setting a watchdog timer that will cause a reset of the CAM if it is no longer descrambling DVB packets properly. There is also a resetCAM() function for doing this directly.
  10. If your device is an S75+ Pro, S75CA Pro, or S75FK Pro, the function setPAT() can be used to cause the receiver to generate Program Association Tables for DVB programs being forwarded from the receiver.
  11. If your device is an S75FK or S75FK Pro, the function setFixedKeys() can be used to program the fixed keys into the receiver.
  12. If your device is an S75 V3 dynamic MAC support, then you can use the function SetDVBMACAddress() to change the DVB MAC.
  13. Use the function getStatus() to collect device status this function relies on passively collected status packets that are being broadcast on the receiver's subnet. Some devices support a pollStatus() function, which allows status to be retrieved from a device not on the same subnet.
  14. Use the function closeS75() to close communications with the device and free the device handle.


Other useful functions include...

  • getPIDList()
  • getProgramList()
  • getPAT()
  • getFixedKeys()
  • getCAMWatchdogTimeout()
  • resetS75()
  • calcSignalStrengthPercentage()
  • getVersion()
