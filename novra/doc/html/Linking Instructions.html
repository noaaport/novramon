Novra Device Communications Library

 

Linux Build

To build a Linux application you need to do the following ...

  • Include the file s75.h in your source file.
  • Put "linux_windows_types.h" in your include path.
  • Either comment out the line where stdafx.h is included in s75.h or include stdafx.h in your include path. MSVC does not allow an #ifdef before an #include "stdafx.h"
  • Define the compile flag "LINUX" either as a definition preceding the #include "s75.h" or in your makefile (ie. CPPFLAGS = -DLINUX)
  • Link to the file libS75.a

Windows Build Using The Static Library

To build a Windows application using the static library you will need to do the following ...

  • Include the file s75.h in your source file.
  • Link to the file S75ComLib.lib.
  • The ws2_32.lib (Winsock 2 library) and the iphlpapi.lib* (IP helper library) will also be needed.

Windows Build Using The DLL

To build a Windows application using the DLL you will need to do the folowing ...

  • Define the compile flag "S75_DLL_EXPORTS"
  • Include the file s75.h in your source file.
  • Link to the import library S75_DLL.lib
  • The ws2_32.lib (Winsock 2 library) and the iphlpapi.lib* (IP helper library) will also be needed.
  • When you run the application the file S75_DLL.dll should either be in your system folder or be a registered DLL.

* The file iphlpapi.lib is the import library for iphlpapi.dll, both of which come with the Microsoft SDK.
